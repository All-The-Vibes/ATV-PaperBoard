03Surface

Input contract.

stable

The atv tier reads the same input shape as the legacy tiers, with one addition: a <code>sections: [&hellip;]</code> array. When present, the section emitters take over and the legacy <code>rows</code> / <code>body_md</code> paths are bypassed.

PropTypeDefaultNotes
titlestringUntitled ArtifactSets <title> and slug. Hero section owns the visible H1.

05Guidance

Design intent.

read once

These rules are not enforced by code. They are how the design holds together.

·Exactly one hero per artifact, at the top.
·Number every section. 00 through NN, two digits.
·Lede paragraphs no longer than three sentences.
·Avoid filling more than 15% of the surface with accent or rose.
·No emojis in section heads. Mono eyebrows only.
·Code shells get a file-path marker when the code targets a specific file.
  • Do not introduce a light theme inside an atv-tier artifact.
  • Do not stack two hero sections.
  • Do not use status-table for non-status data — the badges will mislead.
  • Do not nest sec inside sec beyond two levels.
  • Do not add custom CSS to the rendered HTML. Edit the template.
